# git rebase 常用参数

```
git rebase命令有一些常用的参数，下面是其中一些常见的参数及其作用：
-i, --interactive: 允许以交互方式重新排序、编辑或合并提交。使用此参数可以在变基过程中编辑提交消息、跳过提交、
    #合并提交等操作。
--onto <目标分支>: 指定一个新的基准分支，将当前分支上的提交应用到目标分支之上。
-p, --preserve-merges: 在变基时保留合并提交。默认情况下，Git会尝试简化历史记录，将所有的合并提交转换为单独的提
    #交。使用此参数可以保留原始的合并结构。
-x, --exec <command>: 对每个变基的提交应用指定的命令。可以在变基过程中执行自定义的脚本或命令。
--skip: 在变基过程中跳过当前提交。如果遇到冲突或其他问题无法解决，可以使用此选项跳过当前提交并继续变基操作。
--abort: 中止当前的变基操作，并恢复到变基前的原始状态。
--continue: 在解决完冲突后继续变基操作。
#这些只是一些常见的git rebase参数，还有其他更高级的参数和选项。你可以使用git rebase --help命令查看完整的参
    #数列表以及它们的作用和用法。
```

# git rebase -i 

```
在交互式的 git rebase -i 命令中，你可以对提交进行一系列的操作来重新排序、编辑或合并提交。
下面是一些常见的操作以及它们的具体含义：
pick    : 选择当前提交（默认操作）。代表保留该提交，并将其应用到变基后的分支上。
reword    : 重新编辑当前提交的提交消息。执行此操作后，Git会打开文本编辑器，允许你修改提交消息。
edit    : 暂停变基操作，允许你修改当前提交的内容。执行此操作后，Git会停在该提交上，你可以对文件进行修改、添加、删除等操作。
squash    : 将当前提交与前一个提交合并为一个提交，并将提交消息合并在一起。这个操作可以用来合并多个相邻的提交。
fixup    : 类似于 squash，但丢弃了当前提交的提交消息。这个操作用于将当前提交合并到前一个提交中，而不保留该提交的提交消息。
drop    : 删除当前提交，不将其应用到变基后的分支上。
通过在 git rebase -i 中使用这些操作，你可以对提交进行精细的控制，从而重新组织提交历史、编辑提交消息以及合并
或删除提交。请注意，在 git rebase -i 中修改提交历史可能会改变提交哈希值，因此请谨慎操作，以免影响其他人的工作。
```
